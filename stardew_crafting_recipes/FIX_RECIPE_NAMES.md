# 配方名称汉化问题修复报告

## ✅ 问题已完全解决！

所有配方名称现在都显示为中文了！

## 🐛 问题原因

之前的脚本逻辑：
```python
"名称": custom_name or recipe_name  # ❌ 直接使用英文 recipe_name
```

配方数据中，大部分原版配方的第 6 个字段（自定义显示名称）是空的：
```
"Chest": "388 50/Home/130/true/default/"  ← 最后一个字段为空
```

所以脚本会直接使用 `recipe_name`（"Chest"），导致显示英文。

## ✨ 修复方案

新的逻辑：
```python
# 优先级: 自定义名称 > 产物名称 > 英文配方名
display_name = custom_name if custom_name else (output_info["名称"] if output_info["名称"] else recipe_name)
```

这样，当没有自定义名称时，会使用产物的中文名称作为配方名称！

## 📝 技术细节

配方名称的优先级：

1. **自定义名称**（`parts[5]`）：如果配方数据第 6 字段有值
   - 例如：一些 Mod 配方可能有自定义的中文名称
   
2. **产物名称**（`output_info["名称"]`）：从 BigCraftables 或 Objects 翻译获取
   - 例如：Chest 配方 → 产物是 BigCraftable ID 130 → "宝箱"
   
3. **英文配方名**（`recipe_name`）：作为最后的fallback
   - 仅当前两个都为空时使用
