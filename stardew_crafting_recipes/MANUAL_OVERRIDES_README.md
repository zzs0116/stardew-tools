# 手动覆盖配置说明

## 📝 文件作用

`manual_overrides.json` 用于手动覆盖自动生成的配方数据，方便你自定义特定配方的信息。

## 🔧 使用方法

### 1. 编辑配置文件

打开 `manual_overrides.json`，按以下格式添加覆盖配置：

```json
{
  "配方英文名": {
    "名称": "自定义中文名",
    "描述": "自定义描述",
    "解锁条件": "自定义解锁条件",
    "来源": "自定义来源"
  }
}
```

### 2. 运行更新脚本

```bash
python update_crafting_data.py
```

脚本会自动应用手动覆盖配置。

## 📋 可覆盖的字段

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `名称` | 配方的中文显示名称 | "箱子" |
| `描述` | 配方/产物的描述 | "用于存放物品的容器" |
| `解锁条件` | 解锁方式说明 | "默认解锁" |
| `解锁条件描述` | 解锁条件的详细说明（鼠标悬停时显示） | "在卡利科沙漠的绿洲商店，每次购买价格1000金币" |
| `来源` | 配方来源标签 | "原版"、"SVE" 等 |

> **注意**：字段名必须完全匹配，包括中文字符。
> **提示**：`解锁条件描述` 字段用于提供更详细的信息，鼠标悬停在解锁条件上时会显示。

## 📖 完整示例

假设你想修改 "Warp Totem: Desert" 配方的显示信息：

```json
{
  "_说明": "该文件用于手动覆盖配方数据...",
  
  "Warp Totem: Desert": {
    "解锁条件": "沙漠商人处购买",
    "解锁条件描述": "在卡利科沙漠的绿洲商店，每次购买价格1000金币"
  },
  
  "Chest": {
    "名称": "储物箱",
    "描述": "可以存放各种物品的木制箱子。",
    "解锁条件": "游戏开始即可用"
  },
  
  "Furnace": {
    "来源": "矿井解锁"
  }
}
```

**说明：**
- 以 `_` 开头的字段（如 `_说明`、`_示例`）会被脚本忽略，可用于添加注释
- 只需要覆盖你想修改的字段，其他字段保持自动生成的值
- 配方的 key 必须使用英文名称（`名称_EN`）
- `解锁条件描述` 会在用户鼠标悬停在解锁条件上时显示，用于提供更详细的信息

## 🔍 如何找到配方的英文名称？

打开生成的 `data/crafting_data.json`，查找配方的 `"名称_EN"` 字段：

```json
{
  "名称": "宝箱",
  "名称_EN": "Chest",  ← 使用这个作为 key
  ...
}
```

## ⚡ 工作流程

1. **首次运行**：运行脚本生成初始数据
2. **查看数据**：打开网页或 `crafting_data.json` 查看内容
3. **发现问题**：某些配方显示不正确
4. **添加覆盖**：在 `manual_overrides.json` 中添加修正
5. **重新生成**：再次运行脚本，应用覆盖

## 💡 最佳实践

- ✅ **保持简洁**：只覆盖需要修改的字段
- ✅ **添加注释**：使用 `_注释` 字段记录修改原因
- ✅ **版本控制**：将 `manual_overrides.json` 加入 git 跟踪
- ❌ **避免过度**：大量覆盖说明数据源有问题，应修复数据来源

## 🛠️ 故障排除

### 覆盖未生效

1. 检查配方英文名是否正确（区分大小写）
2. 检查字段名是否正确（必须是中文）
3. 确认 JSON 格式正确（无语法错误）
4. 查看脚本输出是否显示"加载了 X 条手动覆盖配置"

### JSON 格式错误

使用在线工具验证 JSON 格式：https://jsonlint.com/

或者在 VSCode 中打开文件，会自动检查格式错误。
